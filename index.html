検証データ アップロード

モデル検証時のテストデータ（CSVデータ）をアップロードします。

注意：このデータは学習に使用されません

学習モデル定義

TensorFlow jsのLayers API形式でモデル作成のプログラムを入力してください。（入力方法はTensorFlow jsの公式ドキュメントを参照ください。）

const model = tf.sequential(); model.add(tf.layers.dense({units: 1024, activation: 'relu', inputShape: [18]})); model.add(tf.layers.dense({units: 1024, activation: 'relu'})); model.add(tf.layers.dense({units: 1024, activation: 'relu'})); model.add(tf.layers.dense({units: 1024, activation: 'relu'})); model.add(tf.layers.dense({units: 4, activation: 'softmax'})); model.compile({optimizer: 'sgd', loss: 'categoricalCrossentropy', metrics: ['accuracy']}); return model;

注意事項

  • 作成したモデルは最後にcompileを行い、returnで返してください。
  • compile時、必ずmetricsでaccuracyを定めてください。（学習時に精度取得の関数を通過するため、定めなければエラーが発生します。）
  • 現時点では、エラー時にWeb UI上ではコールバックが行われません。（デバッグ時は管理者ツールのコンソールを常時開いておくことをおすすめします。）
  • 入力層数はアーキテクチャ設定、出力層数は目標ラベルセレクタ設定に表示されている数字と必ず一致させてください。（層数が一致しないと学習時にエラーが発生します。）
  • モデルの学習はmodel.fit()を使用します。

目標ラベルセレクタ

必要な出力層数：4

学習時にどの目標ラベル（通称：技コード）を残しておくかを設定できます。（複数選択可）

ポインティング解像度
×

学習データに記録されている座標は、どの解像度の画像に準拠しているものか設定してください。
（最大1920×1080）

エポック数（学習回数）

モデルの学習回数を設定します。（最大2000回）

バッチサイズ（損失関数の範囲）

モデルを評価する際、損失関数に入力されるデータの大きさ（バッチサイズ）を設定します。（最大100）

その他設定

学習時の詳細設定を変更できます。

  • モデルを学習させる際、DevToolsコンソール上に精度などの過程を表示させます。（速度が低下する可能性があります）

  • 実際に予測は行わず、モデルのみを作成します。（モデルのダウンロードは可能です）

※今後、以下の機能を搭載予定です。

・データシャッフル

注意事項

  • 動画処理はローカル上のJavaScriptを使用するため、処理速度は閲覧しているハードウェアに依存します。

  • 本システムは10フレームごとに予測情報などをIndexedDB内に格納しております。長時間の予測を行うと容量不足によるクエリエラーが発生する可能性がございます。（一定数によるデータ解放の機能は後日実装いたします。）

  • 本システムはGPU（WebGL）を使用しています。使用はなるべくグラフィックスに特化したハードウェアを使用してください。

  • PCの性能を最大限発揮するため、 こちらをクリックして使用ブラウザのオプションを『高パフォーマンス』に設定してください。
    （Windowsのみ対応。Macの場合はこちらを参考にGPUの自動切り替えを無効化してください。）
